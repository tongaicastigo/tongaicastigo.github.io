Guia de Instalação Remota de Linux para Estudantes em Moçambique

Olá! Este guia foi criado para o ajudar a instalar o Linux remotamente em computadores com recursos limitados em Moçambique, utilizando uma ligação à internet que pode não ser ideal. A nossa prioridade será a eficiência e a resiliência.

Nota Importante: A instalação remota de um sistema operativo é um processo complexo. Os seus estudantes precisarão de ter um nível básico de familiaridade com o computador e capacidade para seguir instruções precisas. Idealmente, ter alguém no local com conhecimentos técnicos mínimos para auxiliar é crucial.

1. Preparação Pré-Instalação (Em Moçambique)

Antes de começar a instalar, é fundamental que os estudantes façam estas preparações:

  1. Computador Ligado e com Acesso à Internet: O computador deve estar ligado à eletricidade e ter uma forma de se ligar à internet (Ethernet ou Wi-Fi).
  2. Acesso à BIOS/UEFI: Os estudantes devem saber como entrar na BIOS ou UEFI do computador para mudar a ordem de boot. Geralmente, isto envolve premir uma tecla (como F2, F10, F12, DEL) repetidamente ao ligar o computador.
  3. Pen Drive Vazia (mínimo 8GB): Precisaremos de uma pen drive para criar um "live USB" ou um disco de instalação.
  4. Software de Comunicação: Utilize plataformas como WhatsApp, Skype, Google Meet ou Jitsi para comunicação por voz/vídeo e partilha de ecrã (se possível). A comunicação é vital para orientar os estudantes.
  5. Teclado e Rato Funcionais: Assegure-se de que os periféricos básicos funcionam.

2. Escolha da Distribuição Linux Ideal

Para computadores de fracos recursos, a escolha da distribuição é crucial. Recomendo distribuições "leves":

  • Lubuntu: Baseado em Ubuntu, usa o ambiente de trabalho LXQt, que é muito leve.
  • Xubuntu: Também baseado em Ubuntu, mas com o ambiente de trabalho Xfce, que é um pouco mais pesado que o LXQt, mas ainda assim leve.
  • Puppy Linux: Extremamente leve, pode até correr completamente da RAM, ideal para máquinas muito antigas, mas a curva de aprendizagem pode ser um pouco maior.
  • Linux Lite: Projetado especificamente para utilizadores de Windows que transitam para Linux e para computadores mais antigos.
  • AntiX: Uma distribuição muito leve e versátil para computadores mais antigos e com poucos recursos.

Recomendação: Comece com Lubuntu ou Linux Lite, pois oferecem um bom equilíbrio entre leveza e facilidade de uso para quem está a começar.

3. Criação do Live USB Remotamente

Esta é a parte mais complexa e crucial, pois não pode estar fisicamente presente. Terá de guiar os estudantes passo a passo.

  1. Download da Imagem ISO:
    • Peça aos estudantes para descarregar a imagem ISO da distribuição Linux escolhida para o computador deles. Forneça o link direto para evitar confusões.
    • Estratégia para Internet Lenta: Se a internet for muito lenta ou instável, pode ser mais eficiente enviar a ISO por um serviço de partilha de ficheiros que suporte pausas e recomeços (como um cliente BitTorrent para um torrent da ISO, se disponível, ou um gestor de downloads como o `wget` se eles tiverem acesso à linha de comandos no Windows, mas isso é mais avançado).
    • **Alternativa (Se possível):** Se alguém de confiança em Moçambique tiver uma ligação melhor, pode descarregar a ISO e copiá-la para a pen drive.
  2. Software para Criar o Live USB:
    • O software mais recomendado e fácil de usar no Windows é o Rufus ou o Balena Etcher.
    • Guie os estudantes para descarregar um deles (Rufus é mais compacto e direto).
    • Instruções para o Rufus:
      1. Ligar a pen drive ao computador.
      2. Abrir o Rufus.
      3. Selecionar a pen drive em "Dispositivo".
      4. Clicar em "SELECIONAR" e escolher o ficheiro ISO descarregado.
      5. Manter as outras opções como predefinidas (ou pedir para usar as predefinições se o Rufus sugerir diferentes modos).
      6. Clicar em "INICIAR" e confirmar qualquer aviso (avise que os dados na pen drive serão apagados!).
      7. Este processo pode demorar algum tempo. Mantenha-se em comunicação constante.

4. Boot do Live USB e Instalação (Orientação Remota)

Com o Live USB criado, os estudantes devem bootar a partir dele.

  1. Alterar a Ordem de Boot na BIOS/UEFI:
    • Peça aos estudantes para reiniciar o computador.
    • Enquanto o computador reinicia, eles devem premir a tecla correta (F2, F10, F12, DEL, etc.) para entrar na BIOS/UEFI.
    • Navegue-os através dos menus (geralmente usando as setas do teclado) para encontrar a opção "Boot Order" ou "Boot Priority".
    • Mude a ordem para que a pen drive (será listada pelo nome ou como "USB HDD", "Removable Device") seja a primeira opção.
    • Salve as alterações e saia da BIOS/UEFI (geralmente F10). O computador irá reiniciar.
  2. Iniciar o Ambiente Live:
    • Se tudo correr bem, o computador deverá iniciar a partir da pen drive.
    • Irá aparecer um menu com opções. Geralmente, a opção "Try [Nome da Distro] without installing" ou similar é a melhor para testar antes de instalar. Selecione essa opção.
    • Isto carregará o ambiente Linux diretamente da pen drive, sem instalar nada no disco rígido. Isto é excelente para verificar se o hardware é compatível.
  3. Verificação no Ambiente Live:
    • Uma vez no ambiente live, peça aos estudantes para verificar se o Wi-Fi funciona, se o teclado e o rato respondem, e se a resolução do ecrã parece normal.
    • É um bom momento para iniciar uma chamada de voz/vídeo novamente para ter comunicação em tempo real.
  4. Iniciar o Instalador:
    • No ambiente live, haverá um ícone no ambiente de trabalho chamado "Install [Nome da Distro]" ou algo semelhante. Peça para o executarem.
    • O instalador é geralmente gráfico e relativamente intuitivo.
  5. Passos Comuns da Instalação (Guia Remota):
    • Linguagem: Escolher "Português (Portugal)" ou "Português (Moçambique)" se disponível.
    • Layout do Teclado: Escolher "Português (Portugal)" ou o layout apropriado para o teclado que estão a usar.
    • Ligação à Internet: Se o Wi-Fi não foi configurado automaticamente, o instalador pode perguntar para ligar. Ajude-os a selecionar a rede e introduzir a palavra-passe.
    • Tipo de Instalação:
      • "Apagar disco e instalar [Nome da Distro]": Esta é a opção mais simples, mas **AVISE-OS QUE IRÁ APAGAR TUDO NO DISCO RÍGIDO**. Só escolha esta opção se o computador for dedicado ao Linux e não tiver dados importantes.
      • "Instalação personalizada" ou "Something else": Se houver um Windows ou outros dados que querem manter, é preciso ser mais cuidadoso e criar partições manualmente. **Isto é avançado e não recomendado para uma primeira instalação remota com pouca experiência.** Se for absolutamente necessário, terá de os guiar passo a passo na criação de partições para a raiz (`/`), `swap` (memória virtual, se tiverem pouca RAM) e talvez `/home`.
    • Localização: Selecionar a sua localização no mapa ou na lista (Moçambique).
    • Criar Utilizador:
      • Pedir para criar um nome de utilizador e palavra-passe. Escolham uma palavra-passe que consigam memorizar facilmente.
      • Podem escolher a opção de "Iniciar sessão automaticamente" para simplificar, mas é menos seguro.
    • Progresso da Instalação: O instalador irá copiar os ficheiros e configurar o sistema. Este processo pode demorar. Mantenha-se em comunicação.
  6. Reiniciar após a Instalação:
    • Quando a instalação estiver completa, o instalador irá pedir para reiniciar o computador.
    • **IMPORTANTE:** Quando reiniciar, eles devem remover a pen drive USB imediatamente.
    • O computador deverá agora iniciar no novo sistema Linux instalado.

5. Configuração Pós-Instalação e Suporte Remoto

Parabéns, o Linux deve estar instalado! Agora vem a parte de garantir que os estudantes conseguem usá-lo e receber suporte.

  1. Atualizar o Sistema:
    • Uma das primeiras coisas a fazer é atualizar o sistema. Peça para abrir o Terminal (geralmente Ctrl+Alt+T) e digitar os seguintes comandos: 
      sudo apt update
      sudo apt upgrade -y
    • Isto irá descarregar e instalar as últimas atualizações. A internet lenta pode tornar este processo demorado.
  2. Instalar Software Essencial:
    • Navegador web (Firefox já vem instalado, mas podem preferir Chrome/Chromium).
    • Suite de escritório (LibreOffice já vem instalado).
    • VLC Media Player para multimédia.
    • Ferramentas de comunicação (se necessário, como Skype, Jitsi desktop client).
  3. Ferramentas de Acesso Remoto (Importante para o seu Suporte):
    • Para o seu suporte a longo prazo, instale uma ferramenta de acesso remoto nos computadores dos estudantes.
      • TeamViewer: É fácil de usar para ambos os lados, mas pode ser um pouco pesado para computadores antigos e requer uma conta.
      • AnyDesk: Semelhante ao TeamViewer, muitas vezes com melhor desempenho em ligações fracas.
      • Chrome Remote Desktop: Leve e integrado com o Chrome, mas requer que os estudantes tenham uma conta Google e o Chrome instalado.
      • Remmina (Cliente RDP/VNC) + Servidor VNC (TigerVNC/TightVNC): Esta é uma opção mais "Linux-nativa" e leve. Terá de configurar um servidor VNC no lado deles e usar o Remmina no seu lado. É mais técnico, mas muito eficiente.
    • Instruções para configurar o acesso remoto:
      • Escolha uma ferramenta e guie-os na instalação (por exemplo, `sudo apt install teamviewer` se for a sua escolha, ou descarregar o `.deb` do site oficial).
      • Peça para eles partilharem o ID e a palavra-passe para que possa aceder ao computador e dar suporte diretamente.
  4. Educação Contínua:
    • Ensine-os o básico do Linux: navegar no ambiente de trabalho, abrir aplicações, usar o explorador de ficheiros.
    • Ensine-os a usar o Terminal para comandos básicos como `ls`, `cd`, `pwd`, `sudo apt update`, `sudo apt upgrade`.
    • Pode criar pequenos guias em PDF ou vídeo para eles consultarem.

Considerações Adicionais para Moçambique e Internet Lenta

  • Tolerância a Falhas: Prepare-se para que a internet falhe e a instalação seja interrompida. Terá de recomeçar algumas vezes.
  • Download Offline: Se possível, descarregue os .deb (pacotes de instalação) de aplicações essenciais no seu lado e tente transferi-los para os estudantes em formato zip/tar, para que possam instalar offline depois. Isto é mais complexo, mas pode poupar largura de banda.
  • Comunicação Clara e Paciência: Seja extremamente claro nas suas instruções. Use linguagem simples. Tenha muita paciência.
  • Horários: Tente agendar as sessões de instalação para horários em que a ligação à internet em Moçambique tende a ser mais estável (se houver essa informação).
  • Testes Prévios: Se tiver um computador antigo consigo, tente replicar o processo para antecipar problemas.

Este processo será um desafio, mas a sua dedicação em levar o conhecimento de Linux aos estudantes em Moçambique é louvável. Boa sorte!
